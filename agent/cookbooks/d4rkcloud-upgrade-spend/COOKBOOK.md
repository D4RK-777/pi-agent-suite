# D4rkCloud Upgrade Spend

## Purpose

Convert weekly ladder outcomes, D4rkCoin awards, and repeated lessons into actual upgrades in the
right layer of the system.

## Use When

- a weekly winner should claim upgrade priority
- a lane has earned budget through strong work
- repeated pain should fund a focused fix
- the team needs to decide what to improve next instead of vaguely saying "we learned something"

## Inputs

- `docs/D4RKCLOUD_WEEKLY_LADDER.md`
- `docs/D4RKCLOUD_BUDGET_LEDGER.md`
- `docs/AGENT_STRENGTH_SCORECARD.md`
- `docs/CAPABILITY_REFINEMENT_FUNNEL.md`
- evidence from recent tasks, reports, corrections, and wins

## Steps

1. Identify the lane requesting spend.
   - confirm the budget award or priority claim
   - confirm the target weakness or opportunity
2. Use `findings-synthesis` to compress the evidence.
   - what happened
   - why it matters
   - what would improve future results
3. Use `recursive-improvement-loop` to choose the correct destination layer.
   - core skill
   - project overlay
   - agent definition
   - cookbook/process
   - memory note
4. Define the spend item clearly.
   - target artifact
   - scope
   - expected effect
   - verification rule
5. Implement the upgrade.
   - edit the canonical or project-local target
   - update manifests if needed
   - sync downstream surfaces if shared skills changed
6. Record the spend.
   - award used
   - artifact changed
   - expected next-cycle gain
7. Review the result in the next weekly cycle.
   - did the upgrade reduce errors or improve outcomes
   - should the spend model reward similar work again

## Output

- one approved spend item
- one implemented upgrade
- one verification note for the next review cycle

## Rule

Do not approve symbolic spend.

If the spend does not produce a real artifact or a verifiable improvement path, reject it.
