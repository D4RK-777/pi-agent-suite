# Repo Onboarding

## Purpose

Map an unfamiliar repository quickly without over-exploring.

## Use When

- starting work in a new repo
- resuming a repo after context loss

## Steps

1. Identify the entry surfaces, major configs, and route or app roots.
2. Separate live implementation from parked or duplicate code.
3. Locate the files most likely to own the requested behavior.
4. Record the minimum durable map needed for future work.

## Output

- repo map
- live target paths
- open questions that block implementation
