# D4rkCloud Weekly Review

## Purpose

Run a weekly review of the Darclaw stack so the strongest lanes are visible, the weakest lanes are
improved deliberately, and repeated mistakes feed back into the right layer of the system.

## Use When

- you want a weekly Darclaw winner and ladder
- the team wants to compare which agents are actually performing best
- repeated mistakes, corrections, or strong outputs should affect future priorities
- D4rkCoins or upgrade budget should be assigned rationally

## Steps

1. Collect the week's evidence.
   - notable tasks
   - successes
   - user corrections
   - reports
   - recurring failures
2. Score the lanes against the weekly model.
   - outcome quality
   - error rate
   - lane/skill selection quality
   - helpfulness under pressure
   - improvement contribution
3. Use `findings-synthesis` to compress the evidence into one clear ladder.
4. Pick the weekly champion and assign the badge in `docs/D4RKCLOUD_WEEKLY_LADDER.md`.
5. Use `recursive-improvement-loop` to decide which issues should become:
   - core skill improvements
   - project overlay updates
   - cookbook or process updates
   - memory notes
6. Assign D4rkCoins or upgrade budget to the winning lanes.
7. Record what the budget will fund next in `docs/D4RKCLOUD_BUDGET_LEDGER.md`.
8. If spend is approved, run `cookbooks/d4rkcloud-upgrade-spend/COOKBOOK.md` to convert the
   award into a real upgrade.

## Output

- weekly winner
- ranked ladder
- D4rkCoin awards and penalties
- explicit improvement actions for the next cycle
